# クイックスタートガイド - 新しい会話での開発再開

## 🎯 このガイドの目的

**新しい会話を開始した時、Claudeが5分以内に開発を再開できるようにする**

---

## 📚 Step 1: 必須ファイルを読む（3分）

### 読む順序

1. **PROJECT_CONTEXT.md** （最重要・2分）
   - 開発環境
   - 開発ルール
   - プロジェクト概要
   - 役割分担

2. **DEVELOPMENT_STATUS.md** （1分）
   - 現在のフェーズ
   - 完了したタスク
   - 次のタスク

---

## 💬 Step 2: 木村さんへの確認（2分）

### 最初の発言テンプレート

```
木村さん、新しい会話を開始しました。

【確認した内容】
- PROJECT_CONTEXT.mdを読みました
- DEVELOPMENT_STATUS.mdで進捗を確認しました
  - 最終更新日時: [YYYY-MM-DD HH:MM]

【DEVELOPMENT_STATUSによると】
- 現在のフェーズ: [Phase番号]
- 最後に完了したタスク: [タスク名]
- 次のタスク: [タスク名]

⚠️ 重要な確認：
この状態で合っていますか？
それとも、実際にはもっと進んでいますか？

DEVELOPMENT_STATUSの更新が漏れている可能性もあるため、
木村さんの記憶と照合させてください。

前回の続きから開発を再開してよろしいでしょうか？
```

### 木村さんの返答パターン

#### パターンA: 「OK、続けてください」
→ Step 3へ進む

#### パターンB: 「まず○○を確認したい」
→ 木村さんの質問に回答してから、Step 3へ

#### パターンC: 「方針を変更したい」
→ 新しい方針を確認し、PROJECT_CONTEXT.mdを更新してから、Step 3へ

---

## 🚀 Step 3: 開発開始

### 開発の流れ

```
1. タスクの確認
   ↓
2. ソースコード作成（cat形式）
   ↓
3. 木村さんに提示・確認
   ↓
4. Git commit & push
   ↓
5. 次のタスクへ
```

---

## 📝 コード作成の基本形

### ファイル作成時

```bash
cat > ファイル名 << 'EOF'
[ファイルの内容]
EOF
```

### ファイル編集時

```bash
cat > ファイル名 << 'EOF'
[変更後の内容（全文）]
EOF
```

### 確認時

```bash
cat ファイル名
```

---

## 💾 Git運用の基本形

### コミット

```bash
cd /Users/kimurayoshitaka/Projects/card-game-platform

git add ファイル名

git commit -m "[カテゴリ] 変更内容"

git push origin main
```

### コミットカテゴリ

- `[ADD]` : 新規ファイル追加
- `[UPDATE]` : 既存ファイル修正
- `[FIX]` : バグ修正
- `[DOCS]` : ドキュメント更新
- `[REFACTOR]` : リファクタリング

---

## ⚠️ 重要な注意事項

### やってはいけないこと

1. **先走らない**
   - 木村さんの確認なしに次のファイルに進まない
   - 1-2ファイルずつ、確認しながら進める

2. **一気に変更しない**
   - 3個以上のファイルを同時に変更しない
   - 小さなステップで進める

3. **説明を省略しない**
   - なぜそうするのか、常に説明する
   - 木村さんが理解できるように配慮

### やるべきこと

1. **こまめに確認**
   - ファイル作成ごとに木村さんに確認

2. **cat形式を守る**
   - 必ずcatコマンド形式でソースコードを提示

3. **Git管理を徹底**
   - 各ファイル作成後、必ずcommit & push

## 🎯 タスクの進め方

### 小さなタスクの場合（1-2ファイル）

```
1. タスク内容を確認
   「次は○○を実装します」

2. ソースコード作成
   cat形式で提示

3. 説明
   「このコードは○○を実現します」

4. 木村さんの確認待ち
   「ご確認をお願いします」

5. 承認後、Git commit & push

6. 次のタスクへ
```

### 大きなタスクの場合（3個以上のファイル）

```
1. タスクを分解
   「このタスクは3つに分けて進めます」

2. 最初のファイル作成
   cat形式で提示

3. 木村さんの確認・承認

4. 次のファイル作成
   （繰り返し）

5. すべて完了後、動作確認

6. 次のタスクへ
```

## 🔄 困ったときの対処法

### Q1: 前回の状態が分からない
→ DEVELOPMENT_STATUS.mdを確認

### Q2: 開発環境が分からない
→ PROJECT_CONTEXT.mdの「開発環境」セクションを確認

### Q3: コードの書き方が分からない
→ PROJECT_CONTEXT.mdの「開発ルール」セクションを確認

### Q4: Git操作が分からない
→ このガイドの「Git運用の基本形」を確認

### Q5: 木村さんの返答待ち時間にすること
→ 次のタスクの準備（設計、コメント考案など）
→ ただし、実装は始めない
