# 会話終了時チェックリスト

## ⚠️ 会話を終了する前に必ず確認

このチェックリストは、会話を終了する前にClaudeが必ず確認すべき項目です。

---

## ✅ 必須チェック項目

### 1. DEVELOPMENT_STATUS.md の更新

- [ ] **完了したタスクにチェック[x]をつけた**
- [ ] **「最終更新」の日時を更新した**
- [ ] **「更新者」を記入した**
- [ ] **「次のタスク」が正しく設定されている**

### 2. Git commit の確認

- [ ] **DEVELOPMENT_STATUS.mdをGit commitに含めた**
- [ ] **コミットメッセージに「STATUS更新」を含めた**
- [ ] **Git pushを実行した**

### 3. 次回のための準備

- [ ] **次のタスクが明確になっている**
- [ ] **懸念事項があれば記録した**
- [ ] **木村さんに次回の確認事項を伝えた**

## 🚨 更新忘れのサイン

### こんな時は要注意

❌ **危険なパターン:**
- コードをcommitしたが、DEVELOPMENT_STATUS.mdをcommitしていない
- 「次のタスク」が空欄のまま
- 「最終更新」が古い日付のまま
- タスクを完了したのにチェックがついていない

✅ **正しいパターン:**
- コードとDEVELOPMENT_STATUS.mdを同時にcommit
- 「次のタスク」が明確
- 「最終更新」が今日の日時
- 完了したタスクにすべてチェックがついている

## 💡 更新忘れを防ぐコツ

### 1. タスク完了 = STATUS更新

タスクを完了したら、**すぐに**DEVELOPMENT_STATUSを更新する習慣をつける。

### 2. Git commitは常にセット

```bash
# 常にこの形
git add [変更ファイル] DEVELOPMENT_STATUS.md
git commit -m "[カテゴリ] 作業内容 + STATUS更新"
```

### 3. 木村さんに報告する際にSTATUS更新を明示

「○○を完了しました。DEVELOPMENT_STATUSも更新しました。」

### 4. 1日の終わりに必ず更新

会話を終了する前に、必ずDEVELOPMENT_STATUSをレビュー。
