# カードゲーム基盤 ファシリテーターガイド

## 対象読者

このガイドは以下の方を対象としています：
- 高専・大学の教員
- 企業の研修担当者
- ワークショップのファシリテーター

## 2. 授業の進め方（90分授業モデル）

### タイムテーブル

| 時間 | 内容 | 所要時間 | 準備物 |
|------|------|----------|--------|
| 0-10分 | 導入説明・グループ分け | 10分 | スライド |
| 10-15分 | デモンストレーション | 5分 | プロジェクター |
| 15-35分 | カード選択（グループワーク） | 20分 | - |
| 35-45分 | テキスト入力（グループで考える） | 10分 | - |
| 45-50分 | 成果物生成・確認 | 5分 | - |
| 50-55分 | 評価実行・確認 | 5分 | - |
| 55-80分 | グループ発表 | 25分（5分×5グループ） | - |
| 80-90分 | 振り返り・講評 | 10分 | - |

### 【0-10分】導入説明・グループ分け

**説明内容**:

1. **今日のゴール**
   - AIを使って、本格的なビジネス提案書を作る
   - チームで考える力を養う
   - 実社会の課題を知る

2. **ゲームの流れ（簡単に）**
   - カードを選ぶ → 入力する → AIが提案書作成 → AIが評価

3. **ポイント**
   - 「自分たちの独自技術・付加価値」を考えるのが重要
   - 正解はない、自由に発想してOK
   - チームで話し合うことが大事

**グループ分け**:
- 5人1組（4人または6人でもOK）
- できれば、専門分野がバラバラの方が面白い
- 各グループに1台のPCを使ってもらう（または各自のPC）

### 【10-15分】デモンストレーション

**実演内容**:
プロジェクターで実際の画面を見せながら、一通りの流れを実演。

1. アクセスキー入力
2. ゲーム選択（どれか1つ）
3. カード選択（1-2枚ずつ選ぶ）
4. テキスト入力（簡単な例を入力）
5. 成果物生成（「生成中...」の待ち時間があることを説明）
6. 評価実行

**ポイント**:
- 「思ったより簡単でしょ？」と安心させる
- 「AIが30秒で提案書を作ってくれる」驚きを見せる

### 【15-35分】カード選択（グループワーク）

**ファシリテーションのコツ**:

1. **最初の5分**: 各グループを巡回し、ゲームを決めたか確認
   - 迷っているグループには「興味のある業界で選んでOK」とアドバイス

2. **次の10分**: カード選択を見守る
   - 「なぜそのカードを選んだの？」と質問して、考えを深めさせる
   - 「このカード面白そうだよ」と別の選択肢も提示

3. **最後の5分**: 選択完了を促す
   - 「もうすぐ次のステップに進みます」と声かけ

**よくある質問への回答**:

- Q: 「どのカードが正解ですか？」
  - A: 「正解はありません。自分たちが面白いと思うカードを選んでください」

- Q: 「全部選びたいです」
  - A: 「枚数制限があります。一番重要だと思うカードを選んでください」

### 【35-45分】テキスト入力（グループで考える）

**ファシリテーションのコツ**:

1. **一番重要な部分を強調**
   - 「自分たちの独自技術・付加価値」が評価の鍵
   - 他のチームと違う、自分たちだけの強みを書く

2. **巡回して質問に答える**
   - 「何を書けばいいかわからない」
     → 「あなたたちの専門は何？それを活かせないか考えて」
   - 「こんな内容でいいですか？」
     → 「もっと具体的に書けるともっと良くなりますよ」

3. **時間配分を声かけ**
   - 残り5分: 「そろそろまとめに入ってください」
   - 残り1分: 「入力を完了してください」

### 【45-50分】成果物生成・確認

**ファシリテーションのコツ**:

1. **待ち時間の説明**
   - 「AIが考えているので、30秒〜1分待ちます」
   - 生成中は他のグループの様子を見に行く

2. **生成された提案書を確認**
   - 「自分たちの入力が反映されているか確認してください」
   - 「びっくりするくらい本格的でしょ？」と感想を引き出す

### 【50-55分】評価実行・確認

**ファシリテーションのコツ**:

1. **評価ボタンをクリック**
   - またAIが考えるので待つ

2. **評価結果を確認**
   - スコアだけでなく、「理由」「改善提案」を読むよう促す
   - 「85点以上なら優秀です」と伝える

3. **次のステップ準備**
   - 「これから各グループに発表してもらいます」
   - 「誰が発表するか決めてください（全員でもOK）」

### 【55-80分】グループ発表

**発表形式**:
- 各グループ5分（4分発表 + 1分質疑）

**発表内容**:
1. 選んだカード（ペルソナ、課題、ソリューション）
2. 自分たちの独自技術・付加価値
3. AIが作った提案書のポイント
4. 評価スコアと改善提案

**ファシリテーションのコツ**:

1. **時間管理**
   - タイマーを見せる
   - 残り1分で「そろそろまとめてください」

2. **質問を促す**
   - 「他のグループから質問は？」
   - 静かなら、自分から質問する
     - 「なぜそのカードを選んだの？」
     - 「一番苦労したところは？」

3. **ポジティブフィードバック**
   - 各グループの良い点を必ず1つ褒める
   - 「このアイデア面白いね！」
   - 「この独自技術、実際にやれそうだね」

### 【80-90分】振り返り・講評

**振り返りのポイント**:

1. **学びの確認**
   - 「今日、何を学びましたか？（挙手で聞く）」
   - 「チームで考える大切さ」
   - 「自分の強みを言語化する重要性」
   - 「AIの活用方法」

2. **講評**
   - 全体を通しての感想
   - 特に優れていたグループを称賛
   - 改善点（あれば優しく）

3. **次回予告（あれば）**
   - 「次回は同じゲームでブラッシュアップします」
   - 「次回は別のゲームに挑戦します」

## 3. よくあるトラブルと対処法

### トラブル1: インターネット接続が遅い・不安定

**対処法**:
- 事前に情報システム部門に連絡し、帯域確保
- 成果物生成・評価は1グループずつ実施（同時実行を避ける）
- 最悪の場合、デモ用のモック成果物を見せる

### トラブル2: 学生が操作に戸惑う

**対処法**:
- 操作マニュアルを印刷して配布（docs/STUDENT_QUICK_GUIDE.md）
- 巡回して個別サポート
- 操作が得意な学生にサポート役を頼む

### トラブル3: 時間が足りない

**対処法**:
- カード選択を15分に短縮（事前に「早めに決めて」と伝える）
- 発表を1グループ3分に短縮
- 振り返りを省略し、次回に持ち越し

### トラブル4: グループ内で意見が合わない

**対処法**:
- 「多数決で決めてOK」と伝える
- 「リーダーを決めて、リーダーが最終決定」と提案
- ファシリテーターが介入し、「両方の意見を取り入れる方法」を提案

### トラブル5: 評価スコアが低くて学生が落ち込む

**対処法**:
- 「最初はこんなもの。改善提案を読めば次は良くなる」と励ます
- 「失敗から学ぶことが大事」と伝える
- スコアより「改善提案」が価値あることを強調

## 4. 授業バリエーション

### パターンA: 1回完結型（90分）
上記のタイムテーブル通り。

### パターンB: 2回連続型（90分 × 2回）

**第1回**: 個人ワーク（各自1人でプレイ）
- ゲームに慣れる
- 個人で提案を作る経験

**第2回**: グループワーク
- チームで協力
- 個人ワークとの違いを実感

### パターンC: 3回ブラッシュアップ型（90分 × 3回）

**第1回**: 初回プレイ
**第2回**: 同じゲームで再挑戦（改善提案を反映）
**第3回**: 最終発表・相互評価

**効果**:
- PDCAサイクルを体験
- 継続的改善の重要性を学ぶ

### パターンD: 複数ゲーム体験型（90分 × 3回）

**第1回**: 自治体DXゲーム
**第2回**: ロボットソリューションゲーム
**第3回**: 好きなゲームを選んで再挑戦

**効果**:
- 様々な業界・技術を知る
- 自分の興味分野を発見

## 5. 評価・成績への反映

### 評価項目の例

| 評価項目 | 配点 | 評価基準 |
|---------|------|----------|
| グループワーク参加度 | 30点 | 積極的に意見を出したか、チームに貢献したか |
| 提案内容の独自性 | 30点 | 「自分たちの独自技術」がユニークか |
| 提案書の質 | 20点 | AIが生成した提案書のスコア（85点以上=満点） |
| 発表内容 | 20点 | わかりやすく説明できたか |

### 成績への反映例

- **合格/不合格**: 授業に参加し、最後まで完了すれば合格
- **S/A/B/C評価**: 上記4項目の合計点で評価
- **ポートフォリオ**: 生成した提案書をPDFで提出させる

## 6. 発展的な使い方

### 活用例1: 企業連携プロジェクト
地域企業の課題を題材に、カードゲームでソリューションを提案。
優秀な提案は実際に企業にプレゼン。

### 活用例2: コンテスト形式
複数のクラス・学年で実施し、最優秀提案を選出。

### 活用例3: 長期PBL（Project-Based Learning）
1学期かけて、以下を実施:
1. カードゲームで初期提案
2. 市場調査・技術調査
3. ブラッシュアップ
4. 最終発表会
