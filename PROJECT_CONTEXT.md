## 🔧 開発ルール（重要）

### ソースコードの書き方

**必ずcatコマンド形式で書く：**

```bash
cat > ファイル名 << 'EOF'
[ファイルの内容]
EOF
```

**理由：**
1. ファイル全体を一度に確認できる
2. コピー＆ペーストが簡単
3. 差分が明確

### Git運用ルール

1. **1つの変更に対して1-2ファイルずつ変更**
2. **変更後は必ずGitHub Push & Commit**
3. **木村さんに確認してもらってから次へ進む**
4. **⚠️ タスク完了時は必ずDEVELOPMENT_STATUS.mdも更新する**

### タスク完了時の必須ステップ

```
タスク実施
  ↓
木村さんの承認
  ↓
✅ DEVELOPMENT_STATUS.md更新（必須！）
  ↓
Git commit（コードとSTATUSを同時に）
  ↓
完了
```

**重要:** DEVELOPMENT_STATUS.mdを更新しないと、新しい会話で古い状態から再開してしまいます！

### コミットメッセージ形式

```
[カテゴリ] 簡潔な説明

例:
[ADD] 営業支援部門のRFP入力フォーム追加
[UPDATE] 調達支援部門の入力バリデーション修正
[FIX] AIフィードバック生成のバグ修正
[DOCS] README.mdに使い方を追加
```

### アジャイル開発サイクル

**1スプリント = 2時間程度**
- 短いサイクルで開発→確認→改善
- 各スプリントで動作する成果物を作る

## 🎮 プロジェクト概要

### 目的
**設定ファイル（JSON）を変更するだけで、様々なテーマ・業界に対応したカードゲームを提供できる汎用基盤を構築**

### コンセプト
```
【基盤ソフトウェア】
    ↓
設定ファイル（JSON）でカスタマイズ
    ↓
【各種カードゲーム】
- 自治体DX版
- IT運用版
- 営業支援版
- その他
```

### 主要機能
1. **ゲーム選択**: 複数のゲームから1つを選択
2. **カード選択**: カテゴリごとにカードを選択
3. **テキスト入力**: ユーザー独自の情報を入力
4. **AI成果物生成**: Claude APIで成果物を生成
5. **AI評価**: 評価基準に基づいて採点

### ターゲットユーザー
- 企業研修担当者
- 大学教員
- 自治体職員
- コンサルタント
